Example 1: Predict MPG for a Specific Car
──────────────────────────────────────────

Using PowerShell:
─────────────────
$body = @{
    cylinders = 8
    displacement = 400
    horsepower = 150
    weight = 4500
    acceleration = 12
    model_year = 73
    origin = 1
} | ConvertTo-Json

Invoke-WebRequest -Uri http://localhost:8000/predict `
    -Method POST `
    -Headers @{"Content-Type"="application/json"} `
    -Body $body | ConvertFrom-Json

Expected Result:
{
  "predicted_mpg": 12.41,
  "prediction_confidence": "Medium",
  "model_used": "Multilayer Perceptron (MLP)"
}


Using Python (requests):
────────────────────────
import requests
import json

vehicle_data = {
    "cylinders": 8,
    "displacement": 400,
    "horsepower": 150,
    "weight": 4500,
    "acceleration": 12,
    "model_year": 73,
    "origin": 1
}

response = requests.post(
    "http://localhost:8000/predict",
    json=vehicle_data
)

prediction = response.json()
print(f"Predicted MPG: {prediction['predicted_mpg']}")
print(f"Confidence: {prediction['prediction_confidence']}")

🚀 SWITCH BETWEEN MODELS IN API
═══════════════════════════════════════════════════════════════════════════════

The API currently uses MLP. Want to use LSTM (better accuracy)?

1. Open app.py
2. Find the load_model_and_preprocess() function (~line 100)
3. Change the model file:
   FROM: model = tf.keras.models.load_model('models/mlp_model.h5')
   TO:   model = tf.keras.models.load_model('models/lstm_model.h5')
4. Also change the corresponding scaler:
   FROM: 'models/mlp_scaler.pkl'
   TO:   'models/lstm_scaler.pkl'
5. Save and restart the API server

   Typical range: 8-25 seconds (0-60)
   Example: 12

6. model_year
   Typical range: 70-82 (1970-1982)
   Example: 73

7. origin
   Values: 1 (USA), 2 (Europe), 3 (Japan)
   Example: 1

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: API not responding
Solution: Make sure you ran "python app.py" before trying to access it

Problem: Port 8000 already in use
Solution: Either stop the running process or change port in app.py

Problem: "Module not found" error
Solution: Make sure dependencies are installed: pip install -r requirements.txt

Problem: API returns wrong predictions
Solution: Check that the correct model and scaler are loaded in app.py

Problem: Jupyter notebooks won't run
Solution: Install: pip install jupyter ipython

Problem: Can't access http://localhost:8000
Solution: Make sure API server is still running in your terminal
